Coreframe can generate your entire UI component library in a single action. Instead of building component sets by hand, you click one button and Coreframe scaffolds every component, binds it to your design tokens, and organizes everything onto a dedicated page inside your Figma file — ready for your team to start using immediately.

## Generate all components

<Steps>
  <Step title="Generate your design tokens first">
    Make sure you have already run the token generation step. Components bind directly to your Figma variables, so the token collection needs to exist before components are placed. If you haven't done this yet, head to the Tokens tab and generate your token set.
  </Step>
  <Step title="Open the Components tab">
    In the Coreframe plugin panel, click the **Components** tab. You'll see a searchable grid of every available component, organized by category.
  </Step>
  <Step title="Browse and filter the component grid">
    Use the search bar or the category filter pills — Forms, Navigation, Feedback, Data Display, Overlays, and Layout — to explore what's available. Each card shows a live preview thumbnail and the variant count for that component.
  </Step>
  <Step title="Click Generate All">
    Click the **Generate All** button to kick off the full library build. Coreframe queues every component and generates them in sequence.
  </Step>
  <Step title="Coreframe creates a Components page">
    Once generation completes, Coreframe creates (or updates) a page named **Components** in your Figma file and places all generated component sets there in a structured layout, grouped by category.
  </Step>
  <Step title="Review the documentation frames">
    Alongside each component set, Coreframe auto-generates a documentation frame showing the component name, a short description, and a variant overview. These frames are ready to share with developers or stakeholders directly inside Figma.
  </Step>
</Steps>

<Tip>
  Generate tokens first. Components bind to your Figma variables — if variables don't exist, Coreframe creates fallback values, but your components won't update when you change token values later.
</Tip>

## Regenerating components

You can re-run generation at any time — for example, after updating your token values or adding a new color palette. Coreframe finds existing component sets by name and replaces them with freshly generated versions.

<Warning>
  Regenerating will overwrite existing component sets with the same name. Any local overrides or detached instances will not be updated — only the master component set is replaced.
</Warning>

## Component dependencies

Some components are composed from other Coreframe components. For example, a Card may include nested Button, Badge, and Avatar instances, and a Modal may depend on Button and Spinner. Coreframe resolves these dependencies automatically: if a required base component hasn't been generated yet, Coreframe generates it first before building the dependent component. You never have to manage generation order manually.
